## 🔥 第一阶段：地基重修与排雷（第1-6周）
**目标**：把“我以为我会了”变成“我真的测过”。**每天2小时，周末4小时**。

### 第1周：JDK 21 与 虚拟线程——6年经验的你必须拿出干货
- **周一-周二**：看懂虚拟线程原理（不是操作系统线程，是JVM调度的）。资源：[OpenJDK Loom项目Wiki]
- **周三-周五**：**强制任务**——用虚拟线程重写一个老项目中的`CompletableFuture`异步调用。压测对比：传统线程池 vs 虚拟线程。
- **周末**：输出一篇博客/笔记：《我在6年老项目里用虚拟线程把Tomcat QPS提升了X%》。**这是简历里的第一颗子弹**。

### 第2周：JVM急救——不求调优，求“看懂”
- **周一-周三**：看懂G1日志。不要背参数，去生产环境捞一份GC日志，逐行翻译。
- **周四-周五**：用`jmap`+`MAT`分析一次堆Dump（哪怕是自己Demo）。目标：**能找到大对象、内存泄漏嫌疑**。
- **周末**：实战——给一个模拟的OOM场景，写排查报告。

### 第3-4周：并发——把“锁”变成肌肉记忆
- **目标**：面试官问任何并发工具，3秒内说出**适用场景+坑**。
- **强制任务**：手写一个**带超时控制的分布式锁**（Redis + 虚拟线程）。要求：可重入、自动续期、防误删。
- **产出**：GitHub仓库，README写清楚设计文档。**这是简历里的第二颗子弹**。

### 第5-6周：数据库——告别“会写SQL，不会看执行计划”
- **周一-周三**：MySQL索引失效场景实战。找一张百万级表，故意写10种让索引失效的SQL，用`explain`验证。
- **周四-周五**：**MVCC深度**——别背隔离级别。实操：开两个事务，模拟`不可重复读`和`幻读`，看间隙锁怎么解决。
- **周末**：分布式事务（Seata AT模式）。重点不是配置，是**理解“全局锁”和“回滚日志”的实现代价**。

## 🔥 第二阶段：深水区攻坚——构建架构视野（第7-16周）
**目标**：从“会用框架”到“能选型、能排障”。**这是6年经验者的分水岭**。

### 第7-8周：Spring Boot 3 + GraalVM——2026年架构师的标准起手式
- **任务**：用Spring Boot 3.3+写一个纯API服务，**用GraalVM native-image编译成二进制**。
- **坑点**：反射、动态代理在AOT下会炸。你必须学会**用`native-image`配置文件或`@RegisterReflection`**。
- **产出**：Docker镜像，基础镜像用`distroless`，**镜像体积<100MB**。这是大厂简历的硬通货。

### 第9-10周：Kubernetes——不是运维，是架构师的“操作系统”
- **不要**：背yaml。
- **要**：**在本地（Minikube/K3s）部署你的Spring Boot原生镜像**。
- **必须踩的坑**：**存活探针写错导致Pod循环重启、ConfigMap热加载失败**。
- **产出**：Helm Chart包，能一键部署你的应用。面试时直接说“我用Helm管理了3个环境”。

### 第11-12周：可观测性——大厂必考，没人教
- **工具**：OpenTelemetry + Grafana Stack
- **任务**：给你的Spring Boot应用**埋点**（自动+手动）。输出三个面板：
    - 黄金指标（延迟、流量、错误、饱和度）
    - JVM监控（GC、虚拟线程数）
    - 业务监控（订单量、支付成功率）
- **认知升级**：架构师不是等报警，是**定义什么是“正常”**。

### 第13-14周：Quarkus 与 云原生选型
- **任务**：用Quarkus重写一个极简CRUD，**对比Spring Boot Native的启动时间和内存占用**。
- **思考题**：什么场景下你会说服团队放弃Spring Boot选Quarkus？**（Serverless/FaaS/边缘计算）**
- **产出**：一份技术选型对比报告。面试时展示：**“我不是框架粉，我有选型方法论”**。

### 第15-16周：Spring Cloud Alibaba——国内大厂生存工具
- **核心**：Nacos（注册/配置）、Sentinel（限流熔断）
- **深水区**：**Sentinel的滑动窗口算法实现**、**Nacos的CP/AP模式切换代价**。
- **实战**：模拟雪崩场景，用Sentinel做隔离（线程池隔离/信号量隔离）。看效果对比。
